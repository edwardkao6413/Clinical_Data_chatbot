================================================================================
VARIANT 1: ORGANISED METRICS & IMPROVED FILTER USABILITY
================================================================================

OVERVIEW:
This dashboard variant focuses on improving metric organisation and filter panel
usability for the OSVi Biomechanics Database. Key improvements include collapsible
metric grouping, advanced filters, data quality warnings, and recovery trajectory
enhancements.

================================================================================
MODIFICATIONS MADE
================================================================================

1. REORGANISED METRIC SELECTOR (Collapsible Category Groups)
   ─────────────────────────────────────────────────────────

   CHANGED:
   - Previously: Flat metric pill list with group labels
   - Now: Collapsible headers for each metric subcategory

   STRUCTURE:
   - Strength metrics grouped by type:
     * Knee Extension (Isometric, Concentric, Eccentric)
     * Knee Flexion (Isometric, Concentric, Eccentric)
     * H:Q Ratios
     * Ankle Plantarflexion
     * Hip Adduction
     * Hip Abduction

   - Performance metrics grouped by task:
     * DL Countermovement Jump
     * DL Drop Jump
     * DL 10-5 Repeated Hops
     * SL Countermovement Jump
     * SL Drop Jump
     * SL 10-5 Repeated Hops
     * Triple Forward Hop
     * SL Squat

   IMPLEMENTATION:
   - Modified buildGroupedPills() function (line ~813)
   - Each group is now a collapsible section with:
     * Clickable header with chevron icon (▾/▸)
     * Smooth expand/collapse animation
     * Content area containing metric pills

   USER INTERACTION:
   - Click group label or chevron to expand/collapse
   - Selected metrics highlighted in blue
   - All groups start expanded by default

   CSS CLASSES:
   - .metric-group: Container for each group
   - .metric-group-content: Pills wrapper (display:flex)
   - .metric-group-label: Header text
   - .metric-category-header: Top-level category (Strength/Performance)


2. ADVANCED FILTERS (Registration-based data)
   ──────────────────────────────────────────

   ADDED TO POPULATION REPORTING TAB:
   - New collapsible "Advanced Filters" section below basic demographics
   - Five new filter dropdown fields:
     * Competition Level (from demographics.competition_levels)
     * Surgeon (from demographics.surgeons)
     * Graft Type (from demographics.graft_types)
     * Surgical Procedure (from demographics.surgical_procedures)
     * Injury Mechanism (from demographics.injury_mechanisms)

   IMPLEMENTATION:
   - HTML added: Lines ~250-270 (approx)
   - New toggle function: toggleAdvancedFilters()
   - Updated getFilters() function to include new fields:
     * competition_level
     * surgeon
     * graft_type
     * surgical_procedure
     * injury_mechanism

   - Updated init() function to populate new dropdowns via:
     * populateSelect('f-comp-level', demographics.competition_levels)
     * populateSelect('f-surgeon', demographics.surgeons)
     * populateSelect('f-graft-type', demographics.graft_types)
     * populateSelect('f-proc', demographics.surgical_procedures)
     * populateSelect('f-injury-mech', demographics.injury_mechanisms)

   BACKEND INTEGRATION:
   - Filters are passed to /api/query endpoint
   - Backend ignores unknown fields (no changes needed to app.py)
   - Data source: /api/demographics endpoint (already available)

   COLLAPSIBLE BEHAVIOR:
   - Header shows "Advanced Filters" with chevron toggle
   - Starts collapsed to keep basic filters visible
   - Click header to toggle visibility
   - Same styling as Population Filters header


3. DATA QUALITY WARNING BANNER (n < 5)
   ──────────────────────────────────

   ADDED:
   - Prominent warning banner above KPI cards
   - Triggered when filtered population has n < 5 records
   - Only shown when n > 0 (avoids false positives with zero results)

   CSS STYLING:
   - Light red background (#fef2f2)
   - Red border (#fecaca)
   - Dark red text (#7f1d1d)
   - Icon + message layout with flexbox
   - Font size: 13px, moderate padding

   IMPLEMENTATION:
   - Added .warning-banner CSS class
   - Warning logic added to renderReportingTab() function
   - Checks data.n_records after filter application
   - Banner ID: 'pop-warning-banner' (for toggling)
   - Icon: SVG exclamation mark in circle
   - Message: "Low sample size: Results are based on fewer than 5 records.
     Interpret with caution."

   BEHAVIOR:
   - Banner element created once, then shown/hidden on each filter
   - Inserted before KPI container for prominent placement
   - Removed when n >= 5


4. RECOVERY TRAJECTORY TAB FILTERS
   ──────────────────────────────

   ADDED:
   - Surgeon dropdown (optional filter)
   - Graft Type dropdown (optional filter)
   - Both added to the patient form controls

   IMPLEMENTATION:
   - Modified HTML form in TAB 3 (Recovery Trajectory)
   - New form groups added before Load Trajectories button
   - Updated loadRecovery() function to pass filters:
     * surgeon: from recovery-surgeon dropdown
     * graft_type: from recovery-graft dropdown

   BACKEND BEHAVIOR:
   - Filters passed to /api/recovery endpoint
   - Backend currently only accepts: sex, sport, symptom
   - Unknown fields (surgeon, graft_type) are silently ignored
   - UI-side filtering could be added via /api/query if needed

   INIT FUNCTION:
   - Added population of recovery filter dropdowns:
     * populateSelect('recovery-surgeon', demographics.surgeons)
     * populateSelect('recovery-graft', demographics.graft_types)

   NOTE:
   - Backend /api/recovery endpoint may not filter by these yet
   - Future enhancement: implement server-side filtering or client-side
     post-processing using /api/query results
