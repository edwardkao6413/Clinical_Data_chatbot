Clinical Accuracy & Visual Clarity Enhancement

OVERVIEW
--------
This variant builds on the base prototype with clinical threshold indicators
throughout the interface, improving visual clarity and clinical accuracy. The
dashboard now prominently displays reference ranges, warning states, and
clinical decision support cues.

KEY ENHANCEMENTS IMPLEMENTED
=============================

1. CLINICAL RANGES MAPPING OBJECT
   - Location: JavaScript, top of main script
   - Implementation: const CLINICAL_RANGES = { ... }
   - Coverage: All LSI metrics + 24 absolute metrics

   LSI Metrics (Limb Symmetry Index):
     • Green threshold: >= 90
     • Amber (warning): >= 80
     • Red threshold: < 80

   Absolute Metrics (sample):
     • rsidj_DL_: 1.1–1.4 m/s (drop landing RSI, dominant leg)
     • jhDL_: 31–35 cm (jump height, dominant leg)
     • HQcc_L_/HQcc_R_: 0.55–0.65 (hamstring-quadriceps concentric ratio)
     • ReExtIsoL_/ReExtIsoR_: 3.0–4.0 (reactive extension strength ratio)
     • And 19 additional metrics with clinical ranges

   Helper functions:
     • getClinicalRange(metricCode): Returns range object or defaults to LSI
     • classifyValue(value, metricCode): Returns 'good', 'warn', or 'bad'

2. POPULATION REPORTING TAB ENHANCEMENTS

   A. Histogram Charts with Threshold Lines
      • Added chartjs-plugin-annotation@3.0.1
      • LSI metrics: Dual dashed lines at 90% (green) and 80% (red)
      • Non-LSI metrics: Shaded reference range band (green background)
      • Auto-disabled for metrics without defined ranges

   B. KPI Cards — Clinical Color-Coding
      • Updated logic to use classifyValue() against clinical thresholds
      • Green (.kpi.good): Value within optimal range
      • Amber (.kpi.warn): Value in warning zone
      • Red (.kpi.bad): Value below acceptable threshold
      • Maintains existing CSS classes (.kpi.good, .kpi.warn, .kpi.bad)

   C. Summary Statistics Table — Cell Highlighting
      • Mean column: Color-coded for LSI & absolute metrics
      • Median column: Color-coded with lsi-cell styling
      • Uses existing CSS classes: .lsi-good, .lsi-warn, .lsi-bad
      • Non-colored values indicate metric type has no clinical range defined

3. WARNING BANNER FOR SMALL SAMPLES
   • Trigger: n < 5 records after filter application
   • Placement: Below tabs, above KPI cards
   • Style: Amber background (#fef3c7), border (#f59e0b), dark text
   • Message: "⚠ Caution: n = {count}. Results may not be representative."
   • Auto-hides when sample size >= 5
   • Container: #warning-banner-container (dynamically created)

4. RECOVERY TRAJECTORY ENHANCEMENTS (Minimal, Already Present)
   • Existing 90% RTS threshold line is retained
   • Dashed red line at y=90 automatically applied to all LSI trajectories
   • Patient overlay (star markers) aids individual case interpretation

5. METRIC LABELS & ORGANIZATION
   • METRIC_LABELS object unchanged (preserved exactly per spec)
   • Four-tab structure preserved: Population Report | Similar Patients | Recovery | Chat
   • All existing functionality intact

CLINICAL ACCURACY NOTES
=======================

Reference Range Interpretation:
  • Green zone: Patient demonstrates acceptable/normal performance
  • Amber zone: Caution area; monitor closely; may warrant intervention
  • Red zone: Clinical concern; strong consideration for intervention/rehab adjustment

LSI (Limb Symmetry Index):
  • 90%+: Excellent symmetry; RTS-ready in most contexts
  • 80–89%: Acceptable but asymmetry noted; context-dependent RTS decision
  • <80%: Significant asymmetry; consider continued rehabilitation

Absolute Metrics:
  • Ranges reflect normative data for non-injured athletic populations
  • Sport-specific and age-adjusted ranges not yet implemented (future)
  • Classified as 'warn' if within ±15% of range boundaries
